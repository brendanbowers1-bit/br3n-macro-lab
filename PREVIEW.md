# Local Preview Workflow

## Purpose
This repo uses a local-first workflow so visual changes can be reviewed before publishing to GitHub Pages.

## Branch
Work on:

`bfi-local-brand-test`

Do not merge to `main` until the local preview is visually approved.

## Project type
This site is a **Python-built static HTML/CSS site** for GitHub Pages, with a **Next.js dashboard** exported into `reports/publication/dashboard/`.

- **BFI / Macro Lab pages:** generated by `src/bfi_site.py` and `src/site_builder.py` → `reports/publication/`
- **Dashboard:** `web_dashboard/` (Next.js 14, static export)

## Build (required before preview)
From the repo root:

```bash
python scripts/build_site.py
```

This regenerates HTML in `reports/publication/` (including the dashboard static export).

## Local preview — BFI / Macro Lab site
From the repo root:

```bash
cd reports/publication
python3 -m http.server 5173
```

Open:

http://localhost:5173/index.html

Alternative (opens browser automatically on port 8765):

```bash
python scripts/serve_publication.py
```

Open:

http://127.0.0.1:8765/

## Local preview — Next.js dashboard only
For dashboard UI work without rebuilding the full site:

```bash
cd web_dashboard
npm install
npm run dev
```

Open the localhost URL shown in the terminal (typically http://localhost:3000).

## Review checklist
Before publishing, confirm:

- Header branding looks academic and institutional
- No large pasted logo blocks
- No white rectangles behind logos
- No clipped words
- "Bowers Frontier Institute" displays correctly
- "Markets. Machines. Monetary Systems." displays correctly
- Mobile layout works
- Nav does not crowd the logo
- BR3N Macro Lab appears as a lab/division, not the parent brand
- Site still has deep navy/off-white/graphite research-institute aesthetic
- CTAs work
- Dashboards and existing pages still load
- Footer disclaimer remains intact

## Publish rule
Do not merge to `main` or publish to GitHub Pages until the local preview is visually approved.
